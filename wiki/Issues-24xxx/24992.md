Symbolic comparison is wrongly used in several places throughout Sage (overview https://github.com/sagemath/trac_to_gh/wiki/symbolics-nonzero). The ticket replaces code involuntarily calling `__nonzero__` with a call of the `is_zero` or `is_equal` member functions to clarify if simplification is needed or not. The code positions calling `__nonzero__` were obtained by raising an exception in `__nonzero__` and analyzing the resulting doctest fails. Of course not every simplification is unwanted so every case must be considered and tested.

**CC:**  @cheuberg @dkrenn
